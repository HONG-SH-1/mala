읽어봤어. 기획서랑 제미나이 피드백 둘 다 꼼꼼히 봤는데, 솔직하게 말해줄게. 그 전에 핵심이 되는 모델 사양 하나만 사실 확인을 하고 싶어 — 기획 전체가 특정 모델들 위에 세워져 있어서, 그게 실제로 존재하고 사양이 맞는지부터 확인하는 게 중요하거든.검증된 VRAM 제약과 모델 스펙 불일치를 파악했다검증된 VRAM 제약과 모델 스펙 불일치를 파악했다모델 사양 한 가지만 더 확인할게.검증했고 핵심 하드웨어 제약을 발견했으며 현실적 경로를 제시했다검증했고 핵심 하드웨어 제약을 발견했으며 현실적 경로를 제시했다좋아, 솔직하게 갈게. 기획서에 들인 공이 보이고 사고의 방향 자체는 훌륭한데, 제미나이 피드백이 놓친 치명적인 문제 하나가 기획 전체의 토대를 흔들고 있어. 그것부터 짚을게.
가장 큰 문제: 10GB에 35B는 안 올라가 (MoE 메모리 오해)
기획서와 제미나이 피드백 둘 다 같은 전제를 깔고 있어 — "Qwen 3.6-35B-A3B는 활성 파라미터가 3B니까 MoE+4비트로 10GB에 올릴 수 있다." 그런데 이게 사실과 다른 부분이야.
MoE의 핵심 오해가 여기 있어. 26B 파라미터 모델은 추론 시 4B만 활성화되지만, 26B 전체가 메모리에 다 올라가 있어야 해. 활성 파라미터(3B)는 연산 속도를 결정하지 VRAM 점유량을 결정하는 게 아니야. 그래서 검색 결과들이 명시적으로 이렇게 짚어. MoE 모델은 활성 파라미터 수가 시사하는 것보다 더 많은 총 VRAM을 사용한다 — 용량은 메모리로 지불하고, 그 대가로 품질과 속도를 얻는다. Maarten GrootendorstWill It Run AI
실제 숫자를 보면:

Qwen 3.6-35B-A3B (4비트 Q4_K_M): 약 21GB — RTX 4090 24GB나 5090 32GB 급에 들어가는 모델이야. 10GB의 두 배가 넘어. Will It Run AI
Gemma 4 26B (4비트): 약 14~16GB — RTX 3090이나 4090 같은 단일 컨슈머 GPU 범위에 들어가. 이것도 RTX 3080 10GB엔 안 들어가. MindStudio

즉, 기획의 핵심 모델 두 개 다 RTX 3080 10GB에는 올릴 수 없어. docker-compose에 적은 --gpu-memory-utilization 0.7(=7GB)로는 더더욱 불가능하고. 10GB에서 4비트로 현실적으로 굴릴 수 있는 건 Qwen3 14B(~8.3GB Q4)나 8B(~4.6GB Q4)급의 dense 모델, 혹은 Gemma 4 E4B(4B) 정도야. Will It Run AI
그리고 더 중요한 건 — 제미나이는 이걸 못 잡았을 뿐 아니라 "MoE 채택 논리는 완벽합니다"라고 칭찬까지 했어. 제미나이가 제시한 Plan B("일단 Qwen 3.6 하나로 단일 모델 검증")도 그 단일 모델이 35B면 여전히 안 돌아가. 하드웨어 문제를 해결하지 못한 Plan B인 거지.
이건 면접에서 가장 위험한 지점이기도 해. 인프라/백엔드 1티어 면접관이 "MoE인데 왜 VRAM이 부족하죠? 활성 파라미터가 3B인데?"라고 물으면, 이 오해가 그대로 드러나거든. 거꾸로, 여기를 정확히 이해하고 있으면 그 자체가 강력한 어필 포인트가 돼.
제미나이가 맞게 본 것: 오버엔지니어링
이 부분은 제미나이 피드백이 정확해. DVC 버전관리 + GraphRAG 인덱싱 + 멀티에이전트 + LangGraph 순환 + Self-Healing + Claude Code 자동화는 솔직히 별개의 프로젝트 5개를 한 번에 묶은 거야. 신입 포트폴리오로 제한된 기간에 완주하기엔 거의 불가능에 가까워. "일단 마크다운 몇 개 수동 인덱싱으로 Redis 큐 왕복부터 검증하라"는 제미나이의 조언은 옳아. 거기에 하나 더하면 — vLLM과 GGUF(Q4_K_M)와 SGLang 모델 스와핑이 기획서 안에서 섞여 있어. vLLM은 GGUF 지원이 약하고 단일 GPU에서 모델을 갈아끼우는 용도가 아니야. 단일 3080에서 모델 스와핑을 할 거면 llama.cpp나 Ollama가 훨씬 자연스러워. 도구 선택이 손에 익은 상태에서 나온 게 아니라는 신호고, 면접에서 캐물으면 약점이 돼.
내 솔직한 결론
전체적으로 이 문서는 "AI가 그럴듯하게 써준 마스터플랜"의 전형적인 특징을 갖고 있어 — 자신감 넘치는 전문 용어, 면접용 답변 스크립트, 멋진 다이어그램. 그런데 정작 굴려보면 1번 모델부터 하드웨어에 안 올라가. 제미나이가 "S급"이라고 띄워준 게 오히려 독이야. 실제로 못 돌리는 시스템을 들고 가서 면접관이 한 겹만 파면 무너지거든.
방향을 이렇게 잡으면 훨씬 강한 포트폴리오가 돼:

모델 사양을 현실에 맞게 다시 잡아. 3080 10GB라면 Qwen3 14B(dense) 정도로 라우터+분석+코딩 단일 파이프라인을 먼저 돌려. 듀얼 모델 욕심은 그 다음.
"왜 35B가 안 되는지"를 프로젝트의 핵심 서사로 만들어. VRAM 계산, MoE 메모리 vs 컴퓨트 분리, 양자화 트레이드오프를 직접 측정해서 문서화하면 — 그게 제미나이가 말한 PoC보다 훨씬 진짜 "숫자로 아키텍처를 결정하는 엔지니어"의 증거야.
작동하는 작은 시스템 > 못 돌리는 거대한 시스템. Redis 큐 + LangGraph 순환 루프 하나만 진짜로 돌아가도 충분히 좋은 백엔드 포트폴리오야.

혹시 3080 10GB에서 실제로 굴릴 수 있는 모델 조합과 그에 맞춘 현실적인 아키텍처를 다시 짜주는 게 도움이 될까? 아니면 위 VRAM 계산을 면접에서 설명할 수 있게 정리해줄까?
