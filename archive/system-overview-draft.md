이 시스템의 핵심은 "지식(Data) - 두뇌(Model) - 행동(Agent) - 인프라(Infra)"라는 4가지 축이 유기적으로 맞물려 돌아가는 것입니다.
**1. 지식(Data): Obsidian & DVC**
• 
**원천:** 당신의 옵시디언 노트(마크다운)가 시스템의 유일한 지식 근원입니다.  
• 
**관리:** `DVC`와 `Git`을 사용하여, 데이터의 변경 사항을 버전별로 관리하고 필요할 때만 인덱싱하여 시스템 자원을 아낍니다.  
• 
**전략:** 모든 정보를 다 검색하는 대신, `Semantic Chunking`으로 의미 단위를 쪼개고 `엔티티 정규화`를 거쳐 그래프 형태로 저장합니다.  
**2. 두뇌(Model): Specialized SLLM Engine**
• 
**엔진:** `vLLM`을 사용하여 여러 모델을 고속으로 서빙합니다.  
• 
**전략:** 10GB VRAM 제약을 해결하기 위해 **MoE 구조**와 **4비트 양자화**를 사용하여 30B급 성능을 10GB 환경에 구현합니다.  
• 
**분업:** `Gemma 4`(분석/논리)와 `Qwen 3.6`(코딩/구현)을 명확히 분리하여, 각 단계의 최고 효율을 끌어냅니다.  
**3. 행동(Agent): LangGraph & Orchestrator**
• 
**두뇌의 역할:** `LangGraph`라는 관제탑이 질문을 받으면, 그래프 탐색이 필요한지 단순 검색이 필요한지 판단(Router)합니다.  
• 
**협업:** `Redis` 메시지 큐를 통해 분석 에이전트와 코딩 에이전트가 구조화된 JSON 형태로 대화하며 작업을 넘깁니다.  
• 
**자기 치유(Self-Healing):** 코드가 실패하면 Claude Code가 에러 로그를 다시 Orchestrator로 보내서, 시스템이 스스로 오류를 인지하고 수정하는 루프를 돌립니다.  
**4. 인프라(Infra): Docker-Compose**
• 
**철학:** 모든 구성 요소(vLLM, Redis, 에이전트 코드)를 도커 컨테이너로 묶어, 어떤 PC에 가든 `docker-compose up` 한 번으로 동일한 작업 환경을 재현합니다.  
• 
**가용성:** 시스템 중단 시에도 Redis에 상태(State)가 저장되어 있어 마지막 단계부터 재시작 가능합니다.
